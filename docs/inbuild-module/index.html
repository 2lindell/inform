  • inbuild

The functional part of the build manager.

  • Chapter 1: Setting Up

    • Inter Module - Setting up the use of this module.

    • Inbuild Control - The top-level controller through which client tools use this module.

  • Chapter 2: Conceptual Framework

    • Genres - The different sorts of work managed by inbuild.

    • Works - To store, hash code and compare title/author pairs used to identify works.

    • Editions - An edition is numbered version of a work.

    • Copies - A copy is an instance in the file system of a specific edition of a work.

    • Requirements - A requirement is a way to specify some subset of works: for example, those with a given title, and/or version number.

    • Nests - Nests are repositories of Inform-related resources.

  • Chapter 3: Incremental Builds

    • Build Graphs - Graphs in which vertices correspond to files or copies, and arrows to dependencies between them.

    • Build Methodology - Whether to run tasks internally in some merged tool, or run via the shell, or simply trace to the standard output what we think ought to be done.

    • Build Scripts - Scripts are nothing more than list of build steps.

    • Build Steps - A build step is a task which exercises one of the build skills.

    • Inter Skill - A build step is a task such as running inform7 or inblorb on some file.

    • Inform7 Skill - A build step is a task such as running inform7 or inblorb on some file.

    • Inform6 Skill - A build step is a task such as running inform7 or inblorb on some file.

    • Inblorb Skill - A build step is a task such as running inform7 or inblorb on some file.

  • Chapter 4: Managing Genres of Work

    • Kit Manager - A kit is a combination of Inter code with an Inform 7 extension.

    • Extension Manager - An Inform 7 extension.

    • Language Manager - A language is a combination of Inter code with an Inform 7 extension.

    • Project Bundle Manager - A project bundle is a folder holding an Inform 7 work. The app creates these.

    • Project File Manager - A project file is a plain text file of Inform 7 source text.

    • Template Manager - A template is the outline for a website presenting an Inform work.

    • Pipeline Manager - An Inform 7 pipeline.

  • Chapter 5: Services for the Inform Compiler

    • Kits - A kit is a combination of Inter code with an Inform 7 extension.

    • Extension Services - An Inform 7 extension.

    • Extension Dictionary - To maintain a database of names and constructions in all extensions so far used by this installation of Inform, and spot potential namespace clashes.

    • Extension Documentation - To generate HTML documentation for extensions.

    • Extension Census - To conduct a census of all the extensions installed (whether used on this run or not), and keep the documentation index for them up to date.

    • Template Services - An Inform 7 website template.

    • Project Services - An Inform 7 project.

    • Language Services - An Inform 7 language definition bundle.

    • Pipeline Services - A pipeline is a list of steps to be followed by the Inter processor forming the back end of the Inform compiler.

  • Chapter 6: Handling Inform Source Text

    • Source Text - Code for reading Inform 7 source text, which Inbuild uses for both extensions and projects.

    • Headings - To keep track of the hierarchy of headings and subheadings found in the source text.

    • Inclusions - To fulfill requests to include extensions, adding their material to the parse tree as needed, and removing INCLUDE nodes.

    • Control Structures - To specify the syntax of control structures such as repeat, if and otherwise.

    • Virtual Machine Grammar - Grammar for parsing natural language descriptions of a virtual machine.
