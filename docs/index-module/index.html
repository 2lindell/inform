  • index

Indexing tools needed by Inform, as a module. This is version 1.

  • Chapter 1: Starting Up

    • Index Module - Setting up the use of this module.

  • Chapter 2: Indexing

    Utility routines for generating index pages.

    • Index File Services - To provide routines to help build the various HTML index files, none of which are actually created in this section.

    • Documentation References - To enable index or results pages to link into documentation.

    • Lexicon Index - To construct the Lexicon portion of the Phrasebook page of the Index, which gives brief definitions and references for nouns, adjectives and verbs used in source text for the current project.

    • Index Physical World - This section masterminds the creation of the World and Kinds index pages, though it delegates much of the work elsewhere. Though it does belong to core Inform, these indexes will look pretty sparse if the spatial Plugins aren't plugged in.

    • Kinds Index - To produce most of the Kinds page in the Index for a project: the chart at the top, and the detailed entries below.

    • Index Extensions - To keep details of the extensions currently loaded, their authors, titles, versions and rubrics, and to index and credit them suitably.

    • Index Headings - To produce the index contents listing and the XML headings file.

    • Instances - To index instances.

    • Adjectives - To index adjectives.

    • Variables - To index variables.

    • Properties - To index properties.

    • Inferences - To index inferences.

    • Rules - To index rules and rulebooks.

  • Chapter 3: Indexing for Plugins

    • Figures - To produce the index of figures.

    • Sound Effects - To produce the index of sound effects.

    • External Files - To produce the index of external files.

    • Bibliographic Data - To write the Library Card in the index.

    • Spatial - Indexing functions for the spatial structure of the world model.

    • The Player - Indexing the player's initial position.

    • Backdrops - Indexing the player's initial position.

    • Regions - Indexing the player's initial position.

    • The Map - Indexing the player's initial position.

    • Scenes - Parallel to the World index of space is the Scenes index of time, and in this section we render it as HTML.

    • Actions - To produce the index of external files.

    • Commands Index - To construct the index of command verbs.

  • Chapter 4: Visualising Space

    • Spatial Geometry - To deal with vectors and cuboids in a three-dimensional integer lattice.

    • Spatial Map - To fit the map of the rooms in the game into a cubical grid, preserving distances and angles where possible, and so to give each room approximate coordinate locations.

    • HTML Map - To render the spatial map of rooms as HTML.

    • EPS Map - To render the spatial map of rooms as an EPS (Encapsulated PostScript) file.
